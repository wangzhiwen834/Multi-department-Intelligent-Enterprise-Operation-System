# 工作簿容器与日期导航 + 性能优化 设计（子项目 1/4）

> 日期：2026-07-21
> 所属系统：多部门智能经营系统 · 足浴财务模块（v1 已上线）
> 本设计是多部门系统增强计划 4 个子项目中的第 1 个。其余 3 个：① 系统设置（模型切换/主题）② 日期粒度聚合+大屏日期切换 ③ AI 数据抽取管线+定时调度。本子项目为其余子项目提供"按日期组织的工作簿"基础。

## 背景与目标

当前点店铺后直接进入当月工作簿，靠顶部全局 `<input type="month">` 切月（`App.vue:89`）。数据模型已按 `shop + period(YYYY-MM)` 组织工作簿（`workbook` 表 `UNIQUE(shop_id, period)`），但**没有管理/浏览界面**：用户无法一眼看到该店所有月份的工作簿，无法按年组织、无法便捷地在多月间跳转，也无法新建/复制/删除某月工作簿。随着月份累积，"只有一个表"的体感越来越强。

同时工作簿加载已出现"几秒"延迟，且会随内容增多继续恶化，需优化。

**目标**：
1. 在"选店"与"工作簿"之间增加一个**工作簿管理器**容器（年份文件夹树 + 月份详情面板），支持按年/月浏览、打开、新建、从上月复制、删除、显示锁状态。
2. 容器内**日期导航** = 年/月级浏览找表（天/周/年汇总数据查看归子项目 2，不在本子项目）。
3. **优化工作簿加载性能**。

## 范围

**包含**：
- 新增 `WorkbookManager` 视图（三级导航：ShopList → Manager → Workbook）。
- 年份文件夹树 + 月份详情面板 UI。
- 管理操作：打开、新建工作簿（指定任意年/月）、从上月复制表头、删除（软删）、显示锁状态/编辑者。
- 后端：工作簿列表、复制新建、软删除接口。
- `workbook` 表加 `deleted_at` + 索引。
- 性能：gzip + 引导请求合并 + 测量；按需加载工作表作为备选。

**非目标（不在本子项目）**：
- 天/周/年粒度的数据汇总查看与大屏日期切换（子项目 2）。
- AI 数据抽取与大屏数据源（子项目 3）。
- 系统设置/模型切换/主题（子项目 4）。
- 增量保存、协同实时编辑（痛点是加载，不是保存；协同模型不变）。
- 跨月汇总工作簿、年度汇总表（容器只管理月度工作簿；按年分组即天然提供年度组织视图）。
- 无（原"只能创建当月"限制已应用户要求移除：新建工作簿支持指定任意年/月，含历史月与未来月）。

## 整体流程

三级导航，`App.vue` 增加中间态：

```
ShopList(选店) ──pick──> WorkbookManager(该店容器) ──open(period)──> Workbook(具体月)
                                  <──back───
```

- 点店铺卡片进入该店 `WorkbookManager`，不再直接挂表。
- 容器左侧年份文件夹树列出该店所有月度工作簿（period 倒序），右侧选中月份详情面板。
- "打开"某月 → 进入现有 `Workbook` 视图，传入 `shop` + 该月 `period`。
- `Workbook` 的"返回"回到容器（不再回 ShopList）。
- 顶部全局月份选择器在容器内隐藏（`v-if` 条件扩到 manager 态）；容器自带年/月导航。
- `App.vue` 状态流转：`module==='ops' && shop && !openedPeriod` → 显示 Manager；`openedPeriod` 置值 → 显示 Workbook。

## 容器界面设计（布局 B：年份文件夹树 + 详情面板）

### 左栏
- 顶栏：店名 + 业务名；右上「+ 新建工作簿」按钮（点击展开年/月选择器，默认当月）。
- 树：
  - 年份节点（▼2026 / ▶2025），可折叠，默认展开当前年。
  - 展开后是该年各月（1月…12月），**只列出已存在工作簿的月份 + 当前日历月**。
  - 当前日历月高亮。
  - 月份节点带锁小圆点：该月工作簿正被编辑时显示（数据来自列表查询的 `lockedBy` 聚合）。
- 选中某月 → 右栏详情。

### 右栏（详情面板）
- 标题：`X月工作簿` + 锁徽标（复用现有三态：编辑中 / 某人编辑中 / 查看）。
- 元信息：最近保存时间（`workbook_snapshot.updated_at`）。
- 操作区：
  - **打开工作簿**（主按钮）
  - **从上月复制表头**（仅当选中月份存在上月工作簿时可用；把上月表头复制到选中月份）
  - **删除**（二次确认弹窗）
- 空态：该店无任何工作簿 → 引导「新建当月」。

### 锁状态来源
- 树节点锁点：列表查询 `GET /api/shops/:shopId/workbooks` 一次带出每个 workbook 是否有活跃锁（`sheet_lock` JOIN 聚合）。
- 详情面板锁徽标：复用现有 `GET /api/workbooks/:id/locks/:sheetKey`。

## 后端接口

| 接口 | 方法 | 说明 |
|---|---|---|
| `/api/shops/:shopId/workbooks` | GET | 返回该店工作簿列表 `[{id, period, status, updated_at, lockedBy}]`，period 倒序，排除软删。`lockedBy` 为活跃锁持有者用户名或 null。 |
| `/api/workbooks` | POST | 现有，upsert by `shop_id+period`，用于"新建当月"（从模板构建）。 |
| `/api/workbooks/:id/copy-from` | POST | 从上月复制新建：取 `fromPeriod` 上月 snapshot，保留表头与表单结构（格式/公式/布局），清空所有已填数据，不结转期初，存为本月（`period`）snapshot。 |
| `/api/workbooks/:id` | DELETE | 软删除：设 `deleted_at`，从树隐藏。snapshot 与已同步的 `daily_metric`/`expense` 保留（大屏数据不受影响）。有活跃锁时拒绝。 |
| `/api/workbooks/:id/locks/:sheetKey` | GET | 现有，详情面板锁状态复用。 |
| `/api/workbooks/bootstrap` | POST | 性能用：一次返回 `{wbId, template, snapshot, lockStatus}`。`snapshot` 仅含活动表 cellData + 全量 styles + 全表元信息，非活动表 cellData 不返回（按需加载）。 |
| `/api/workbooks/:id/sheets/:sheetKey/celldata` | GET | 按需加载：返回指定工作表 cellData（styles 复用 bootstrap 已返回的全量集）。 |

### "从上月复制新建"语义
- **只复制表头与表单结构**（列头、单元格格式、公式、合并、列宽行高、布局），**清空所有已填业务数据**，**不结转期初余额**。
- 本月以"空白表单"起步。
- 若上月表头与模板一致，效果近似从模板新建；保留复制路径是为了继承员工对表头/布局的实际调整。
- 目标月份 = 用户在年/月选择器中指定（任意年/月）；若该月工作簿已存在则提示并禁用创建。

## 数据模型变更（最小）

- `workbook` 表加 `deleted_at TIMESTAMPTZ NULL`（软删除）。
- 索引：`CREATE INDEX idx_workbook_shop_period_live ON workbook(shop_id, period) WHERE deleted_at IS NULL;`（树列表查询）。
- 其余复用现有：`workbook_snapshot`（1:1）、`sheet_lock`、`daily_metric`、`expense` 不动。

## 性能优化（分层：先低风险高回报，不够再加）

**根因核实（已读现有代码）**：
- 后端**未开 gzip**（`server/src/index.ts` 仅 `express.json({limit:'10mb'})`，无 `compression`）。
- 加载链**5 次顺序往返**：`me() → createWorkbook(POST) → template(GET) → getSnapshot(GET) → lockStatus(GET)`，全 `await` 串行（`Workbook.vue:193-204`）。云部署每次往返都有延迟，这是"几秒"主因之一。
- snapshot **已是稀疏存储**：`cellData` 只存非空单元格（`sheet-io.ts` 导入 `includeEmpty:false`；`MAX_ROWS=400` 只是渲染上限，不存空行）。→ "砍空行"无用，不做。

**方案**：
1. **开 gzip**：加 `compression` 中间件，snapshot 等大 JSON 压缩传输。
2. **合并引导请求**：新增 `POST /api/workbooks/bootstrap?shopId=&period=` 一次返回 `{wbId, template, snapshot, lockStatus}`，把 4 次往返压成 1 次。`me()` 与 `template` 无依赖可并行。
3. **按需加载工作表**（必做，用户明确要求"点进去再渲染"）：
   - 打开工作簿时 `createWorkbook` 只填充活动表 `cellData`，其余表建空壳（有名称/维度/列宽行高/合并，无 cellData）。
   - 切到某标签时，若该表未加载，调 `GET /api/workbooks/:id/sheets/:sheetKey/celldata` 取其 `cellData` 注入；`styles` 已在 bootstrap 全量返回，样式 id 直接复用，保住单元格样式。
   - 已加载的表缓存，不重复请求；接管重载（`takeoverPoll` 内 `mountUniver()`）同样走 bootstrap 活动表优先。
4. **测量**：上 1-3 后记录 snapshot 体积、首屏耗时、切表耗时（日志/埋点）。

**实现待确认点**：Univer 注入工作表 `cellData`（带样式）的批量 API（候选：`SetRangeValuesCommand` 接受 cell 矩阵，或 worksheet facade 的 cellData setter）。实现计划阶段做短 spike 验证；若不便保样式，回退为"切表时 `setValues` 写值 + 复用已加载 styles 单独套用"。

**明确不做**：砍空行（已稀疏）、增量保存（痛点是加载不是保存）。

## 边界与错误处理

- **复制新建时上月不存在** → 按钮禁用 + 提示"无上月工作簿可复制"。
- **删除正被他人编辑的工作簿** → 拒绝（有活跃锁不可删，返回 409 + 提示持有者）。
- **软删后再建同月** → upsert 复用原行并清 `deleted_at`（snapshot 保留则复用，否则从模板/复制构建）。
- **两人同时"从上月复制新建"同月** → `UNIQUE(shop_id, period)` 兜底，后者返回 409。
- **当前月已存在** → "新建当月"/"从上月复制新建"禁用，详情面板直接提供"打开"。
- **删除仅软删**：snapshot、`daily_metric`、`expense` 保留，大屏历史数据不受影响；如未来需物理清理另立任务。
- **权限**：复用现有 RBAC。删除/新建按现有工作簿写权限校验逻辑（不引入新的角色规则）。

## 测试策略

- **后端 vitest**：
  - 列表接口：按 period 倒序、排除软删、`lockedBy` 聚合正确。
  - copy-from：表头/结构保留、数据清空、期初未结转、上月不存在报错。
  - delete：软删（`deleted_at` 置值）、有锁拒绝、大屏数据（`daily_metric`）未删。
  - bootstrap：活动表 cellData + 全量 styles + 全表元信息；非活动表 cellData 缺省。
  - per-sheet celldata：返回指定表 cellData 正确。
- **前端**：
  - `WorkbookManager` 组件：树渲染、年份折叠、空态、选中→详情、操作流转（打开/新建/复制/删除）。
  - 锁徽标三态显示。
  - 按需加载：首屏只渲染活动表；切表触发拉取与注入、不重复请求；接管重载走活动表优先。
  - 与 `App.vue` 三级导航流转（shop→manager→workbook→manager）。

## 依赖与后续

- 本子项目无前置依赖，是其余 3 个子项目的基础（它们读取"按日期组织的工作簿"）。
- 后续子项目 2（日期粒度+大屏切换）将复用本子项目的工作簿列表与 `daily_metric`；子项目 3（AI 抽取）读取工作簿 snapshot；子项目 4（系统设置）独立。
- 本子项目实施"按需加载工作表"：子项目 3（AI 抽取）读取工作簿时应直接读后端 snapshot/celldata，不依赖前端已加载的表（前端按需加载不改变后端全量数据）。
