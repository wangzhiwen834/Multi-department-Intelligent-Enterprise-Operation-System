# 日期粒度聚合 + 数据大屏重构 — 设计规格

> 子项目 2。本文件是实现契约:后端聚合接口、前端大屏、类型定义都以本规格为准。
> 实施计划见 `docs/superpowers/plans/2026-07-21-日期粒度与大屏重构.md`。

## 1. 背景与问题
- `daily_metric.metrics`(JSONB)已由模板 `daily_ops` 存有一套与《经营报表》行对齐的丰富指标键:`revenue / cash_revenue / customers_total / customers_member / customers_group / customers_walkin / new_members / recharge_total / recharge_first / recharge_renew / recharge_gift / member_consume / therapist_wage / therapist_attendance / total_clocks / clocks_arranged / clocks_requested / clocks_added / footbath_revenue / spa_revenue / minor_revenue`。`reconciliation` 表的支付渠道键 `cash / douyin / meituan / pos / alipay / wechat` 通过 sync 的 `metrics || $4` 合入同一日的 `metrics`。
- 但 `Dashboard.vue` 只可视化其中一小撮(营收/客流/充值/业务结构/支付/费用),且只支持月粒度(`period=YYYY-MM`)。
- 本子项目:后端加 日/周/月/年 粒度聚合 + 趋势下钻;大屏按真实报表维度重画;删任务仪表盘;为子项目 3 的 AI 抽取铺好数据契约。

## 2. 已确认设计决策
1. **粒度切换**:日/周/月/年。大屏自带切换器 + ‹ › 翻页 + 「回到今天」。Dashboard 自管日期状态,默认 `granularity=month, date=今天`。
2. **趋势图 = 下钻到更细粒度**(KPI 用所选区间合计,趋势用区间内更细一级):
   - month -> 当月每日营收(折线)
   - week -> 当周 7 天每日营收
   - year -> 当年 12 个月营收(柱)
   - day -> 往前 14 天每日营收(单天无内部细粒度,trailing 提供上下文)
3. **任务/时间进度**:大屏彻底移除。数据层 `shop.monthly_target` 与报表进度行保留(AI 仍可查),仅大屏不展示。免除周/日粒度下目标按比例拆分。
4. **多店**:保留门店筛选 + 营收排名(按所选区间汇总,不含 target/taskProgress)。指标维度取自大河坎报表,适用于所有足浴店。
5. **模板**:本子项目不扩模板,只用现有指标键。模板/工作簿行扩展留给子项目 3。

## 3. 后端聚合规则
- **接口**:`GET /api/dashboard/overview?granularity=day|week|month|year&date=YYYY-MM-DD&shopId=<可选>`
  - `granularity` 缺省 `month`;`date` 缺省今天;`shopId` 缺省 = 全部店汇总。
  - **向后兼容**:若传 `period=YYYY-MM` 且无 `date`,映射为 `granularity=month, date=period + '-01'`。
- **区间计算**(pure JS date math,服务端 Node,Date 可用):
  - day -> `[date, date]`
  - week -> `[ISO 周一, 周日]`(周一 = `date - ((getDay()+6) % 7)` 天,即 JS `getDay()` 周日=0 转换;或等价 `date_trunc('week', date)` 语义)
  - month -> `[月初, 月末]`
  - year -> `[1/1, 12/31]`
- **KPI 聚合**:`SELECT SUM((metrics->>'key')::numeric)` over `daily_metric WHERE date BETWEEN $start AND $end [+ shopFilter]`。
  - 加性指标 SUM。
  - `avgCustomerPrice = Σrevenue / Σcustomers_total`(重算,非求和)。
- **趋势查询**:同一区间(day 除外)按更细一级 `date_trunc` 分组求和 `revenue`;day 用 `[date-13, date]` 逐日。
  - label:day/week/month -> `MM-DD`;year -> `M月`。
- **门店排名**:`SELECT shop, SUM(revenue) ... WHERE date BETWEEN $start AND $end GROUP BY shop ORDER BY revenue DESC`(忽略 shopId 过滤,总是全店对比)。
- **费用 Top**:`SELECT subject, SUM(amount) FROM expense WHERE pay_date BETWEEN $start AND $end [+ shopFilter] GROUP BY subject ORDER BY amount DESC LIMIT 10`。
- **查询数**:4(KPI+结构一次 sum 多 key、费用、排名、趋势)。`paymentChannels / businessStructure / customerStructure / clockStructure / rechargeStructure` 都从 KPI 那条 sum 查询取(同一行多 key)。

## 4. 后端 payload 契约(替换现有 DashboardOverview)
```ts
interface DashboardOverview {
  granularity: 'day' | 'week' | 'month' | 'year';
  date: string;            // YYYY-MM-DD(请求的锚点日)
  rangeStart: string;      // YYYY-MM-DD
  rangeEnd: string;        // YYYY-MM-DD
  shopId: number | null;
  kpis: {
    revenue: number;             // 营业收入 Σ
    customersTotal: number;      // 总客流 Σ
    avgCustomerPrice: number;    // Σrevenue/Σcustomers 重算
    rechargeTotal: number;       // 充值总额 Σ
    totalClocks: number;         // 总钟数 Σ
    newMembers: number;          // 新增会员 Σ
    therapistAttendance: number; // 技师出勤(人天)Σ
    therapistWage: number;       // 技师工资 Σ
    memberConsume: number;       // 会员消费 Σ
  };
  revenueTrend: { label: string; revenue: number }[];
  customerStructure: { member: number; group: number; walkin: number };
  clockStructure: { arranged: number; requested: number; added: number };
  rechargeStructure: { first: number; renew: number; gift: number };
  businessStructure: { footbath: number; spa: number; minor: number };
  paymentChannels: { cash: number; douyin: number; meituan: number; pos: number; alipay: number; wechat: number };
  expenseBySubject: { subject: string; amount: number }[];
  shopRanking: { shopId: number; shopName: string; revenue: number }[];
}
```
- 删除旧字段:`taskProgress / timeProgress / period`。
- 数值统一 `Number(...)` 转换;空区间返回 0/空数组(不报错)。

## 5. 前端 Dashboard.vue 布局
- **顶栏**:标题「静水楼台·经营驾驶舱」+ 区间标签 + 粒度分段控件(日/周/月/年)+ ‹ › + 「回到今天」+ 门店筛选 + 实时同步 pill。
  - 区间标签:day -> `2026-07-15 周三`;week -> `2026-07-13 ~ 07-19`(ISO 周一~周日);month -> `2026 年 7 月`;year -> `2026 年`。
  - ‹ › 步长:day ±1 天 / week ±7 天 / month ±1 月 / year ±1 年。
  - 切换粒度时 `date` 保持不变(锚点日落在新区间内)。
- **KPI 卡(2 行 × 4)**:营业收入、总客流、平均客单价、充值总额、总钟数、新增会员、技师出勤(人天)、技师工资。
- **图表(8)**:
  1. 营收趋势(line;year 用 bar)
  2. 客流结构(pie:会员/团购/散客)
  3. 钟数结构(stacked bar:排钟/点钟/加钟)
  4. 充值结构(pie:首充/续充/赠送)
  5. 业务结构(pie:足浴/SPA/小项)
  6. 支付渠道(pie)
  7. 费用 Top(bar)
  8. 门店营收排名(bar,点击柱子钻取选店)
- **删除**:任务仪表盘卡 + 5 店任务表 + `taskPct/timePct/behind/paceNote/gauge/gaugeOffset/taskRows` + 「任务完成」KPI 卡。
- 沿用 `--od-*` token、`THEMES['light']`、`Chart.vue`、`money()` 格式化。
- `fetch()` 调 `api.dashboardOverview(granularity, date, shopId)`;`watch([granularity, refDate, shopId], fetch)`。

## 6. App.vue 改动
- `<Dashboard v-model:period="period" />` -> `<Dashboard />`(自管日期)。
- 顶栏 `<input type="month">` 的 `v-if` 由 `!(module === 'ops' && shop)` 改为 `!(module === 'ops' && shop) && module !== 'dashboard'`(dashboard 模块隐藏全局月份选择器)。Chat/ShopList 仍用顶栏月份,不受影响。

## 7. 边界与正确性要求
- 空区间(无数据):KPI 全 0、trend 空数组、structure 全 0、ranking 仍返回全店(营收 0)、expense 空数组。HTTP 200。
- 周界:week 用 ISO 周(周一始)。跨月/跨年周按 ISO 归属。
- `shopId=null`:KPI/structure/expense/trend 按「全部店汇总」(SUM 跨店);ranking 总是全店。
- 向后兼容:`period=YYYY-MM` 必须仍工作(AI 工具 `get_kpis` 等不依赖本接口,但保留以防调用方)。
- `date` 非法/缺省 -> 用今天;`granularity` 非法 -> 400 或降级 month(实现选 400 更安全)。
