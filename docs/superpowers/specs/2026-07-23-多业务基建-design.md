# 多业务基建 + 公司经营切换 — 设计(spec)

- 日期:2026-07-23
- 范围:多业务扩展 4 子项目中的 **子项目 1**(基建)。不碰数据大屏。
- 状态:已与用户确认设计,待写实现计划。

## 1. 背景

当前系统整套数据结构为**足浴专用**:`footbath.template.ts` 三张表(经营报表/收入对账/费用明细)列全是足浴指标(技师工资/总钟数/会员消费…);`dashboard.routes.ts` 的 `KPI_KEYS`、支付结构、业务结构也写死足浴。

用户要新增 4 个业态(汉庭酒店、禧悦月子会所、GALA酒吧、培训学校),并支持业务自定义增删。经评估,这套需求跨多子系统,已分解为 4 子项目:

| # | 子项目 | 依赖 |
|---|---|---|
| **1(本spec)** | 多业务基建 + 公司经营切换 | 无 |
| 2 | 大屏泛化 + 业务下拉切换(样板:酒店)+ AI 抽取按业态适配 | 1 |
| 3 | 其余业态复制样板(月子/酒吧/培训各自模板 + 抽取 prompt) | 2 |

> 原计划「大屏分屏(一屏并排多业务)」已取消:用户澄清「分屏」= 每业务一个独立大屏、下拉切换,即子项目 2 的业务下拉。

**数据模型方向(用户已定)**:各业态独立指标模板——每个业态有自己的表(张数不确定)与指标列,大屏按业态读对应模板显示。模板内容属子项目 2/3,本 spec 不涉及。

## 2. 目标 / 非目标

### 目标(子项目 1)
- 业务(business)增删改查;新建业务**不带模板**(空模板占位)。
- 侧边栏「公司经营」改为可展开导航组,子菜单列出全部业务 + 「全部业务」入口,实时随业务增删变化。
- 业务卡片网格保留,数据源改为 `GET /api/businesses`(0 门店业务也显示)。
- 业务 logo 数据化:`business.logo_path` 存 img src 路径,卡片按它显示;支持上传/删除 logo。
- 硬删级联(物理删该业务下全部门店/工作簿/指标/费用/模板),仅董事长,带审计 + 前端二次确认。

### 非目标(留给后续子项目)
- 各业态模板内容(酒店/月子/酒吧/培训的三表+指标列)→ 子项目 2/3。
- 大屏泛化(从写死足浴改成按业务模板动态选指标)+ 数据大屏业务下拉 → 子项目 2。
- AI 抽取按业态适配 → 子项目 2。
- 新业务门店开工作簿录入数据:本子项目下,新业务门店开工作簿应返回「模板待定义」(因无模板),引导至子项目 2。

## 3. 数据模型

仅一处 schema 变更(追加到 `schema.sql`):

```sql
-- business 加 logo_path:存 img 的 src 路径(可空)
ALTER TABLE business ADD COLUMN IF NOT EXISTS logo_path TEXT;
```

- `business` 现有 `id, code, name, created_at` → 加 `logo_path`。
- **不**给 business 加 `status`:删除=硬删(物理删行),无软删标记(与门店软删不同)。
- `shop`/`template`/`workbook`/`daily_metric`/`expense` 结构不变。
- seed:`UPDATE business SET logo_path='/footbath-logo.png' WHERE code='footbath'`——把现有足浴 logo 纳入数据驱动,前端不再 hardcode。

### logo_path 取值约定
`logo_path` 直接存 `<img src>` 用的路径字符串:
- 仓库静态资源(足浴):`/footbath-logo.png`(随 `web/public` 走,不需要 token)。
- 用户上传:`/api/uploads/business-logos/<file>`(走公开静态目录)。
卡片渲染:有 `logo_path` → `<img :src="logo_path">`;否则通用占位 SVG。

## 4. 后端 API

新建 `server/src/business/business.routes.ts`,挂到 `/api`。写操作均 `requireRole('chairman')` + `auditLog`,沿用 `settings.routes` 套路。

| 方法 | 路径 | 角色 | 说明 |
|---|---|---|---|
| GET | `/api/businesses` | 全员 | 列全部业务,返 `id, code, name, logo_path, shop_count`(`shop_count`=该业务下**活跃门店**数 `status='active'`;硬删=物理删,故无需 status 过滤) |
| POST | `/api/businesses` | 董事长 | body `{name}`;`code` 自动生成 `b${Date.now().toString(36)}`;`logo_path=null`;**不建 template** |
| PATCH | `/api/businesses/:id/rename` | 董事长 | body `{name}` |
| DELETE | `/api/businesses/:id` | 董事长 | 硬删级联(事务,见 §6) |
| POST | `/api/businesses/:id/logo` | 董事长 | body `{image(base64 data URL), originalName}`;存文件到 `uploads/business-logos/`,更新 `logo_path`,返回业务行(base64→磁盘文件套路同海报 logo) |
| DELETE | `/api/businesses/:id/logo` | 董事长 | 清空 `logo_path` 并删磁盘文件 |

### 静态资源
`index.ts` 增加:
```ts
const bizLogosDir = path.join(config.uploadsDir, 'business-logos');
fs.mkdirSync(bizLogosDir, { recursive: true });
app.use('/api/uploads/business-logos', express.static(bizLogosDir));
```
`<img src>` 不带 Bearer token,走公开静态(与现有 `/api/uploads/logos` 海报 logo 同款)。

### 校验
- name:zod `string().trim().min(1).max(80)`;空 → 400(validation)。
- logo image:base64,大小上限沿用海报 logo 10mb;非 base64/超大 → 400。
- `:id` 非整数 → 400;业务不存在 → 404。

### 注册路由
`index.ts` `import { businessRouter } from './business/business.routes.js';` + `app.use('/api', businessRouter);`。

## 5. 前端

### 5.1 侧边栏(`App.vue`)
「公司经营」由扁平项改为可展开导航组:
- onMounted 调 `GET /api/businesses` 拉业务列表供子菜单。
- 点「公司经营」标签 → 切换展开/收起(**不**直接跳页)。
- 子菜单项:`全部业务` + 各业务名。
  - 点「全部业务」→ `module='ops'`, `selectedBusiness=null` → ShopList 卡片总览(状态 A)。
  - 点某业务 → `module='ops'`, `selectedBusiness=<code>` → ShopList 直接进该业务门店列表(状态 B),跳过卡片网格。
- 业务增删改后 ShopList `emit('businesses-changed')` → App.vue 重拉业务列表,子菜单实时同步。
- 侧边栏收起态下子菜单不展开(收起时点「公司经营」直接走总览,避免溢出)。
- 该「点标签展开子菜单 + 业务项」结构供子项目 2 给「数据大屏」加同款业务子菜单复用。

### 5.2 ShopList 改造
- **状态 A 卡片网格**:数据源从「按 shops 反推」改为直接 `GET /api/businesses`(0 门店业务也显示)。每卡显示 logo(`logo_path`?`<img>`:通用 SVG)+ 业务名 + `shop_count`。
- **新增业务卡**(董事长,网格末尾,与「新增门店」卡同款):行内输入业务名 → `POST /api/businesses` → 刷新 + `emit('businesses-changed')`。
- **每卡管理操作**(董事长):重命名(行内)、上传/删除 logo、删除业务(弹「输入业务名确认」二次校验,因硬删级联不可逆)。
- 点卡片(非管理区)→ 进该业务状态 B。
- **状态 B 门店列表**:沿用现有门店增删改;面包屑加返回「全部业务」;按选中业务 code 客户端过滤 `shops`。

### 5.3 类型与 API 客户端
```ts
// types.ts
interface Business { id:number; code:string; name:string; logo_path:string|null; shop_count:number }
// api.ts
businesses():                            GET    /api/businesses
createBusiness(name):                     POST   /api/businesses
renameBusiness(id,name):                  PATCH  /api/businesses/:id/rename
deleteBusiness(id):                        DELETE /api/businesses/:id
uploadBusinessLogo(id,image,originalName):POST   /api/businesses/:id/logo
deleteBusinessLogo(id):                   DELETE /api/businesses/:id/logo
```

## 6. 级联删除(硬删)

`DELETE /api/businesses/:id` 在**一个事务**里按依赖顺序物理删:

```sql
BEGIN;
-- 1) 工作簿(snapshot / sheet_lock 靠 ON DELETE CASCADE 自动带走)
DELETE FROM workbook WHERE shop_id IN (SELECT id FROM shop WHERE business_id=$1);
-- 2) 指标、费用
DELETE FROM daily_metric WHERE shop_id IN (SELECT id FROM shop WHERE business_id=$1);
DELETE FROM expense     WHERE shop_id IN (SELECT id FROM shop WHERE business_id=$1);
-- 3) 门店
DELETE FROM shop     WHERE business_id=$1;
-- 4) 模板
DELETE FROM template WHERE business_id=$1;
-- 5) 业务本身
DELETE FROM business WHERE id=$1;
COMMIT;
```

不改各表外键为 CASCADE(避免影响门店日常软删等其他路径),由 handler 显式按序删。任一步失败 ROLLBACK,整体不变。

### logo 文件清理
删业务前先取 `logo_path`;事务提交后,若它指向 `/api/uploads/business-logos/<file>` 则删磁盘文件;`/footbath-logo.png` 这种仓库静态资源不删。`DELETE /:id/logo` 同样删对应磁盘文件。

## 7. 安全

硬删级联不可逆,三道兜底:
1. **角色**:`requireRole('chairman')`,非董事长 403。
2. **审计**:`auditLog('business.delete'/'create'/'rename'/'logo.upload'/'logo.delete')` 记录操作者与目标。
3. **前端二次确认**:删业务弹窗要求**手输该业务全名**匹配后才发请求(比删门店更危险)。

边界:删足浴这种已有大量数据的业务也允许(用户选的硬删级联即干净清除);二次确认 + 审计防误触。

## 8. 测试

新建 `server/test/business.test.ts`,沿用 `helpers.resetDb` + supertest + boss/mgr 账号套路。

- **GET /api/businesses**:返全部业务,含 `logo_path` 与 `shop_count`(0 门店业务也出现)。
- **POST /api/businesses**:董事长建→201、`code` 自动、`logo_path=null`、无 template 行;经理→403;空名→400;建后该业务门店开工作簿→400「模板待定义」(验证空模板占位;该错误信息由 workbook 路由改中文)。
- **PATCH /:id/rename**:改名成功;经理→403;不存在→404。
- **DELETE /:id(级联)**:建业务→建 2 店→各建工作簿+插指标+费用→删业务:事务后 `business/shop/workbook/daily_metric/expense/template` 全 0 行(workbook_snapshot/sheet_lock 靠级联清);经理删→403 且数据不动;不存在→404。
- **logo 上传/删除**:上传 base64→文件落 `uploads/business-logos/`、`logo_path` 指向它、GET 取到;删 logo→`logo_path=null`、磁盘文件删;经理上传→403。
- **回归**:既有 `shop.test.ts` 等自带种子,不受影响。

前端沿用项目惯例(无前端单测):`vue-tsc --noEmit` + `vite build` 兜类型与编译;人工点一遍侧栏展开/新增业务/上传 logo/删业务。
