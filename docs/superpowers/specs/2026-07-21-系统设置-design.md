# 系统设置(模型管理 + 深浅色主题)- 设计规格

子项目 4。本文件是实现契约:模型列表/分配、AI 网关读 DB、主题切换、前端设置页都以本规格为准。

## 1. 背景与问题

v1 的 AI 三处模型都写死在 `server/.env`:`DOUBAO_MODEL`(对话分析)、`POSTER_MODEL`(文生图海报)、`EXTRACT_MODEL`(AI 抽取,默认复用 DOUBAO_MODEL)。换模型要改 .env 重启,且员工/经理看不到也改不了。主题方面:应用只有浅色一套(`tokens.css` 的 `--od-*` 只在 `:root`),`theme.ts` 虽定义了 dark/light/warm 三套 ECharts 配色但 Dashboard 写死 `THEMES['light']`,无切换入口。

本子项目做两件事:
- **A. 模型管理**:ccswitch 式的模型列表 + 按 AI 功能(分析/海报/抽取)分配模型,管理员 UI 操作,DB 持久化,网关读 DB(env 兜底)。
- **B. 深浅色主题**:全局 light/dark 切换(CSS 变量 + ECharts 跟随),localStorage 记住。

## 2. 已确认设计决策

> 标 `[待确认]` 的评审时定稿;下文按推荐方案写。

1. **模型列表来源 = 自动拉取 Ark `/models`**(已实测可行):`GET {baseUrl}/models` 返 126 模型,按 `status` 过滤掉 `Shutdown`/`Retiring` 得可用集,再按 `task_type` 分类(`TextGeneration`->chat/extraction,`TextToImage`->poster,其余 video/3D/embedding 排除)。设置页「刷新模型列表」按钮触发拉取,结果缓存进 `ai_model` 表。手动添加作为兜底(列表里没有但能用的)。**已确认**(探测脚本 `scripts/probe-ark-models.ts` 验证 200 + 36 可用 + 当前两模型在列)。
2. **存储 = DB 两张小表**:`ai_model`(模型列表)+ `ai_feature_config`(功能->模型)。多用户共享、重启不丢、可审计。不用 config 文件。`[待确认]`
3. **功能粒度 = 3 个**:chat(分析对话)/poster(文生图)/extraction(AI 抽取)。每个分配一个模型。抽取默认复用 chat 的模型(向后兼容)。`[待确认]`
4. **配置权限 = chairman 专属**:系统设置是管理动作,仅董事长可改;经理/员工只读当前分配(用于显示)。`[待确认]`
5. **主题 = light/dark 两套**:满足深浅需求;`theme.ts` 的 warm 暂不在切换器里(留扩展)。`[待确认]`
6. **网关读 DB + env 兜底**:`getFeatureModel(feature)` 先查 `ai_feature_config`,无则回退 `config.doubaoModel`/`posterModel`/`extractModel`。未配 DB 时行为同现状。
7. **主题持久化 = localStorage**(`theme` key),不上 DB(个人偏好,非系统配置)。

## 3. Part A:模型管理

### 3.1 数据模型(schema.sql 追加)

```sql
CREATE TABLE IF NOT EXISTS ai_model (
  id SERIAL PRIMARY KEY,
  model_id TEXT NOT NULL UNIQUE,        -- Ark 模型 id,如 doubao-seed-2-1-pro-260628
  label TEXT NOT NULL,                  -- 显示名(Ark 返回的 name 字段)
  kind TEXT NOT NULL DEFAULT 'chat',    -- chat(文本生成) | image(文生图) | other(视频/3D/embedding,不可分配)
  status TEXT,                          -- Ark 返回的 status(Active/Retiring/Shutdown;拉取时只存非 Shutdown/Retiring)
  task_type JSONB,                      -- Ark 返回的 task_type 数组,留作展示
  fetched_at TIMESTAMPTZ,               -- 上次从 Ark 拉取时间
  enabled BOOLEAN NOT NULL DEFAULT TRUE,
  created_at TIMESTAMPTZ DEFAULT now()
);
CREATE TABLE IF NOT EXISTS ai_feature_config (
  feature TEXT PRIMARY KEY,             -- chat | poster | extraction
  model_id TEXT NOT NULL REFERENCES ai_model(model_id) ON DELETE RESTRICT,
  updated_at TIMESTAMPTZ DEFAULT now()
);
```
- `kind` 由 `task_type` 映射:含 `TextToImage`/`ImageToImage` -> `image`;含 `TextGeneration`/`VisualQuestionAnswering` -> `chat`;其余 -> `other`。
- seed:用 env 的 3 个模型(doubao-seed-2-1-pro / doubao-seedream-4-0 / 同对话模型给 extraction)插入 + 3 条 feature_config,保证开箱即用。首次「刷新」后由 Ark 数据覆盖/补全。

### 3.2 网关改造(ai.gateway.ts + poster.routes.ts)

新增 `getFeatureModel(feature: 'chat'|'poster'|'extraction'): Promise<string>`:
- 查 `SELECT model_id FROM ai_feature_config WHERE feature=$1`。
- 命中 -> 返回;未命中/出错 -> 回退 env 默认(chat->doubaoModel,poster->posterModel,extraction->extractModel)。
- 进程内 5s 缓存(避免每次调用都查库;改分配时清缓存)。

改造调用点:
- `callDoubao(messages)`(对话,带 tools):`model: await getFeatureModel('chat')`。
- `callDoubaoJson(messages)`(抽取):`model: await getFeatureModel('extraction')`。
- `poster.routes` 的 `/poster/generate`:`model: await getFeatureModel('poster')`。
- `callDoubao`/`callDoubaoJson` 改为 async 取 model(已经是 async,无影响)。

### 3.3 路由(settings.routes.ts,chairman 专属)

挂 `authRequired` + `requireRole('chairman')`(除只读 GET)。
- `GET /api/settings/ai` -> `{ models: AiModel[], features: { chat, poster, extraction } }`(所有登录用户可读,供前端显示当前模型)。
- `POST /api/settings/ai/models` body `{ model_id, label, kind }` -> 新增(auditLog `settings.ai.model.add`)。
- `DELETE /api/settings/ai/models/:id` -> 删除(若被 feature_config 引用则 409,auditLog)。
- `PUT /api/settings/ai/features/:feature` body `{ model_id }` -> 分配(auditLog `settings.ai.feature.assign`)。
- `POST /api/settings/ai/models/refresh` -> 调 `GET {baseUrl}/models`(Bearer key),过滤 `status` 非 Shutdown/Retiring,按 `task_type` 定 `kind`,upsert 进 `ai_model`(刷新 `fetched_at`),返回更新后的模型数。失败(Ark 不可达/未配 key)返 502 + 错误。auditLog `settings.ai.model.refresh`。
- `POST /api/settings/ai/models` body `{ model_id, label, kind }` -> 手动兜底新增(auditLog `settings.ai.model.add`)。

### 3.4 前端设置页(Settings.vue,chairman 专属)

- 侧栏加"系统设置"项(仅 chairman 可见,与 employees/audit 同级条件)。
- 页面两块:
  - **模型列表**:顶部「刷新模型列表」按钮 -> 调 refresh,显示 `fetched_at`。表格列 model_id / 显示名 / 类型(chat/image/other)/ 状态 / 操作(删除)。"添加模型"(手动兜底:model_id + 显示名 + 类型)。
  - **功能分配**:3 行(chat/poster/extraction),每行一个下拉(从 enabled 模型里按 kind 过滤:poster 只能选 image/universal,chat/extraction 选 chat/universal),选中即 PUT。
- `GET /api/settings/ai` 加载;变更后刷新。非 chairman 不显示侧栏项。

### 4.2 主题状态(theme-store.ts)

新建 `web/src/composables/theme-store.ts`(响应式单例):
```ts
export type ThemeKey = 'light' | 'dark';
const theme = ref<ThemeKey>(localStorage.getItem('theme') === 'dark' ? 'dark' : 'light');
const apply = (k: ThemeKey) => { document.documentElement.dataset.theme = k; localStorage.setItem('theme', k); };
export const useTheme = () => {
  const toggle = () => { theme.value = theme.value === 'dark' ? 'light' : 'dark'; apply(theme.value); };
  return { theme, toggle };
};
```
- App.vue `onMounted` 调 `apply(theme.value)`(初始应用,避免刷新闪烁可加内联脚本,但 MVP 接受首帧浅色)。
- Dashboard.vue:`const { theme } = useTheme(); const echartsTheme = computed(() => THEMES[theme.value] ?? THEMES.light);` 替换写死的 `THEMES['light']`。watch theme -> Chart 自动 resize 重绘(Chart.vue 已 watch theme)。

### 4.3 App.vue:顶栏切换按钮

顶栏(od-top-right)加一个图标按钮(☀/🌙),`@click="toggle"`。`theme` 变化时按钮图标跟随。

## 5. 接口契约(汇总)

| 方法 | 路径 | 鉴权 | 用途 |
|---|---|---|---|
| GET | /api/settings/ai | authRequired | 读模型列表+功能分配(所有登录用户) |
| POST | /api/settings/ai/models | chairman | 新增模型 |
| DELETE | /api/settings/ai/models/:id | chairman | 删除模型(被引用 409) |
| PUT | /api/settings/ai/features/:feature | chairman | 分配功能->模型 |
| POST | /api/settings/ai/models/probe | chairman | (可选)拉取 Ark 候选模型 |
