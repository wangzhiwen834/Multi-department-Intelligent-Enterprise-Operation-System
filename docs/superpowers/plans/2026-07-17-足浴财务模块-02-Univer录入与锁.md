# 足浴财务模块 · 02 Univer录入+悲观锁+同步 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现在线表格录入面:前端 Univer 编辑 + 后端悲观锁(工作表级)+ 快照持久化 + 模板驱动同步(录入数据进 PG)+ 资金台账自动生成。

**Architecture:** 前端 Vite+React+Univer Sheets Preset 编辑工作簿;保存时前端按 `template.definition` 把录入表提取成结构化行发给后端,同时存整份快照(持久化/重载)。后端做:模板管理、工作簿/快照存取、**悲观锁服务(占/续/放/接管)**、同步(校验+upsert `daily_metric`/`expense`+算合计)、资金台账(滚动结余)。本期单编辑者并发,无 OT。

**Tech Stack:** (后端)延续 01:Express+pg+zod+vitest;(前端,02b)Vite+React+TS+`@univerjs/preset-sheets-core`+Tailwind。

## Global Constraints

- 锁粒度:工作表级 `(workbook_id, sheet_key)`
- 浏览不锁,点"编辑"才占锁;15s 心跳续约;60s 宽限自动释放;可接管;保存前校验锁归属防覆盖
- 录入归一:经营报表/收入对账/费用明细人工填;仅"合计"后端自动求和;资金台账自动生成只读
- 模板驱动:加指标只改 `template.definition`,不改代码
- 前端提取结构化行 -> 后端校验+upsert(不依赖服务端 Univer)
- 频繁提交

## Task 3: 悲观锁服务(核心,必做好)

**Files:** `server/src/lock/lock.service.ts`, `server/src/lock/lock.routes.ts`, `server/test/lock.test.ts`
**Interfaces:** `acquireLock(wbId, sheetKey, user)`、`heartbeat(wbId, sheetKey, user)`、`releaseLock(...)`、`takeover(...)`;`POST/PUT/DELETE /api/workbooks/:id/locks/:sheetKey`、`POST .../takeover`

**规则:**
- acquire:无人持锁或锁已过期(`expires_at < now`) -> 占;否则返回当前持有者
- heartbeat:`UPDATE expires_at = now()+30s WHERE user_id=me`(续约;若锁已转交则 409)
- release:`DELETE WHERE user_id=me`(若不属我则 409)
- takeover:仅当持有者 `expires_at < now()`(已超时)才可强制接管
- 锁记录:`acquired_at`, `expires_at`(=acquired+30s,心跳每 15s 续;宽限 60s 体现在"过期即被接管"——续约间隔 15s,过期窗口 30s,断线后约 30s 可被接管)

- [ ] **Step 1: 写 lock.service.ts**(上述四函数,用 `query` + 事务/条件 UPDATE)
- [ ] **Step 2: 写 lock.routes.ts**(占/续/放/接管 REST)
- [ ] **Step 3: 写 lock.test.ts**(占锁成功/他人占用返回持有者/心跳续约/过期后被接管/释放校验归属)
- [ ] **Step 4: 跑测试通过**
- [ ] **Step 5: Commit** `feat(02a): 工作表级悲观锁(占/续/放/接管)`

## Task 4: 同步(校验+upsert+算合计)

**Files:** `server/src/sync/sync.service.ts`, `server/src/sync/sync.routes.ts`, `server/test/sync.test.ts`
**Interfaces:** `POST /api/workbooks/:id/sync { dailyMetrics:[{date,metrics:{...}}], expenses:[{...}] }` -> 校验模板 + upsert `daily_metric`/`expense` + 返回 `autoSums`(每个 entry 指标的月合计)+ 校验错误列表

**逻辑:**
- 取工作簿绑定的 template.definition
- dailyMetrics:每行按 template 的 `daily_ops`+`reconciliation` 列校验类型;upsert `daily_metric`(shop_id,date,metrics JSONB merge)
- expenses:按 `expense` 列校验;插入 `expense`(带 source_workbook_id)
- autoSums:对每个 `kind=auto_sum` 列,按其所属 sheet 的 entry 指标求月合计(如 `revenue_total = sum(revenue)`)
- 返回 `{ autoSums, errors }`

- [ ] **Step 2: 写 sync.routes.ts**
- [ ] **Step 3: 写 sync.test.ts**(同步后 daily_metric 可查;autoSums 正确;非法类型进 errors)
