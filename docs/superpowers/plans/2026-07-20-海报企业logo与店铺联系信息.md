# 海报模块：企业 Logo + 店铺联系信息

## 目标
在「AI 每日海报」模块支持：
1. 上传 / 删除企业 logo（可多个，永久保存到服务端磁盘 + 数据库；后续可持续新增）
2. 按店铺预设保存地址 + 联系电话（文字）
3. 海报编辑时，把 logo 图片与当前店铺的地址 / 电话作为图层放到画布上

## 已确认的关键决策
- 地址 / 电话 = **按店铺各一份**（挂在 shop 表）。
- logo = **企业级全局共享**（不关联具体店铺/business），因为海报里「店铺」是选填项，logo 是品牌级素材，且现阶段一个企业主体。

## 数据模型（改 `server/src/db/schema.sql`，全部用 IF NOT EXISTS，可安全重跑）
```sql
ALTER TABLE shop ADD COLUMN IF NOT EXISTS address TEXT;
ALTER TABLE shop ADD COLUMN IF NOT EXISTS phone TEXT;

CREATE TABLE IF NOT EXISTS poster_logo (
  id SERIAL PRIMARY KEY,
  filename TEXT NOT NULL,        -- 磁盘存储名(随机 id + 扩展名)
  original_name TEXT NOT NULL,   -- 用户上传时的原始文件名
  mime TEXT NOT NULL,
  size INT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);
```
执行方式不变：`npm run migrate`（整体跑 schema.sql）。

## 后端

### 静态文件服务（`server/src/index.ts`）
- 顶层挂载 `app.use('/api/uploads/logos', express.static(logosDir))`，**不走 authRequired**（`<img src>` 不带 Bearer token；logo 是公开展示的企业标识）。
- 启动时 `fs.mkdir(logosDir, { recursive: true })` 确保目录存在。
- 该路径自动走 vite proxy（开发）+ Nginx `/api/` 反代（生产），**无需改 Nginx / DEPLOY**。

### `server/src/config.ts`
- 加 `uploadsDir`（默认 `path.resolve(process.cwd(), 'uploads')`，可被 `UPLOADS_DIR` 覆盖）；logosDir = `uploadsDir/logos`。

### `server/src/poster/poster.routes.ts`（已有 authRequired + zod + auditLog）
- `GET /api/poster/logos` → 列出全部 logo，每条附 `url = /api/uploads/logos/{filename}`。
- `POST /api/poster/logos` `{ image: dataUrl, originalName }` → 校验 mime ∈ {png,jpeg,webp} + 解码后 < 2MB → `crypto.randomUUID()` 生成文件名 → 写磁盘 + 插库 → 返回新 logo。挂 `auditLog('poster.logo.upload')`。
- `DELETE /api/poster/logos/:id` → 查记录 → 删磁盘文件 + 删库 → `{ ok }`。挂 `auditLog('poster.logo.delete')`。
- 上传走 base64 JSON（与现有 `/poster/generate` 返回 dataUrl 一致），**不加 multer 依赖**。

### `server/src/shop/shop.routes.ts`
- `GET /shops` SELECT 增加 `s.address, s.phone`。
- 新增 `PATCH /shops/:id` `{ address?, phone? }` → 仅更新这两字段 → 返回更新后 shop。

## 前端

### `web/src/types.ts`
- `Shop` 加 `address?: string | null; phone?: string | null;`。
- 新增 `interface Logo { id; filename; original_name; mime; size; url; created_at }`。

### `web/src/api.ts`
- `posterLogos()` / `posterUploadLogo(image, originalName)` / `posterDeleteLogo(id)`。
- `updateShopContact(id, { address, phone })` → PATCH `/api/shops/:id`。

### `web/src/views/Poster.vue`（配置区新增两块卡片，放在「海报配置」卡片下方）
- **企业 Logo 卡片**：缩略图网格 + 上传按钮（`<input type=file>` → `FileReader.readAsDataURL` → POST）+ 每个 logo 的删除按钮（二次确认）。`onMounted` 加载列表。
- **店铺联系信息卡片**：列出所有店铺，每店一行 = 地址输入 + 电话输入 + 保存按钮。
- 把 `logos`、当前选中店铺的 `shopAddress` / `shopPhone` 通过 props 传给 `PosterEditor`。

### `web/src/views/PosterEditor.vue`（Fabric 画布，扩展工具栏）
- props 新增 `logos`、`shopAddress`、`shopPhone`。
- 工具栏新增 **「Logo」按钮**：展开 logos 缩略图，点某个 → `FabricImage.fromURL(url)` 加到画布左上角，按画布宽度 25% 等比缩放，可拖动 / 缩放 / 删除，`role='logo'`。
- 工具栏新增 **「联系信息」按钮**：若已选店铺且有值，把地址（上）+ 电话（下）作为两个 IText 加到底部居中，`role='address'` / `'phone'`；未选店铺或无值时禁用并提示。
- `resetSeedTexts` 不补回 logo / address / phone（这些是用户主动添加的，不自动重建）。

## 验证步骤
1. `cd server && npm run build && npm run migrate`（建表 / 加列），`npm run dev` 启动。
2. `cd web && npm run dev`，登录进入「AI 每日海报」。
3. 上传 pho1 / pho2 为 logo → 删除一个 → 再传；给某店填地址电话并保存。
4. 生成海报 → 点「Logo」把 logo 加到画布 → 点「联系信息」把地址电话加上 → 调整位置 → 下载，确认导出 JPEG 含 logo 与文字。

## 明确不做
- 不加 multer（base64 上传，与现有海报图一致，零新依赖）。
- 不改 Nginx / DEPLOY.md（`/api` 反代已覆盖静态文件）。
- 不写自动化测试（项目测试聚焦财务核心，海报模块无现成测试；以手动验证为准）。
- logo 不按 business 区分（全局共享，适配「店铺选填」）。
