# 工作簿容器与日期导航 + 性能优化 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在"选店"与"工作簿"之间加一个工作簿管理器（年份文件夹树 + 月份详情面板），支持按年/月浏览/打开/新建/复制/删除月度工作簿并显示锁状态；同时用 gzip + 引导请求合并 + 工作表按需加载优化打开速度。

**Architecture:** 三级导航 ShopList -> WorkbookManager -> Workbook。后端新增列表/软删/复制上月/bootstrap/按表取数据接口；`workbook` 表加 `deleted_at` 软删列。性能：后端开 gzip + `POST /workbooks/bootstrap` 一次返回活动表 cellData+全量 styles+元信息，前端 `createWorkbook` 只渲染活动表，切标签按需拉取注入（保存前先补全所有表，防止覆盖未访问表的数据）。

**Tech Stack:** 后端 Node.js + Express + TypeScript + pg + zod + vitest/supertest；前端 Vue3 + Vite + TypeScript + @univerjs/presets（1.0.0-alpha.3）+ Tailwind。

## Global Constraints

- 后端测试：`cd server && npm run test`（vitest run，单进程串行）；测试库由 `TEST_DATABASE_URL` 指定（`server/test/setup.ts` 把 `DATABASE_URL` 指向它）。测试用 `supertest` 打 `app`，每个 `beforeEach` 调 `resetDb()`（TRUNCATE 全表 RESTART IDENTITY）。
- 本地 PG：`localhost:5432`，user=`postgres` pwd=`123456`，库 `enterprise_ops`（开发）+ `enterprise_ops_test`（测试）。`config.databaseUrl = required('DATABASE_URL')`（`server/src/config.ts:12`），故可用 `DATABASE_URL=... npm run migrate` 内联覆盖迁移目标库。
- 前端无单元测试框架；前端任务以 `cd web && npm run build`（`vue-tsc --noEmit && vite build`）作为类型/编译验证，辅以手动核对清单。
- 快照结构：Univer `IWorkbookData` = `{ name, styles: Record<id, IStyleData>, sheetOrder: string[], sheets: Record<sheetId, { id, name, rowCount, columnCount, cellData: Record<row, Record<col, { v, s? }>>, mergeData, columnData, rowData }> }`。`cellData` 稀疏，`s` 为 styles 表的 id 字符串。

- 提交信息沿用仓库中文风格（`feat:`/`fix:`/`docs:`）。

在 `server/src/workbook/workbook.routes.ts` 顶部 `CreateBody` 之后加辅助函数：

```ts
// 复制上月快照:每表只保留第 0 行(表头)cellData,清空其余行;保留 styles/sheetOrder/列宽/行高/合并
const headersOnlySnapshot = (snap: any): any => {
  const sheets: Record<string, any> = {};
  for (const sid of Object.keys(snap.sheets || {})) {
    const sh = snap.sheets[sid];
    const cellData = sh.cellData || {};
    const kept: Record<string, any> = cellData['0'] ? { 0: cellData['0'] } : {};
    sheets[sid] = { ...sh, cellData: kept };
  }
  return { ...snap, sheets };
};
```

    if (!srcSnap) return res.status(400).json({ error: '上月工作簿无快照' });
    const transformed = headersOnlySnapshot(srcSnap.data);
    await query(
      `INSERT INTO workbook_snapshot (workbook_id, data, updated_at) VALUES ($1,$2,now())
       ON CONFLICT (workbook_id) DO UPDATE SET data=EXCLUDED.data, updated_at=now()`,
      [id, JSON.stringify(transformed)],
    );
    await query('UPDATE workbook SET updated_at=now() WHERE id=$1', [id]);
    res.json({ ok: true });
  } catch (e) { next(e); }
});

const activeOnlySnapshot = (snap: any): any => {
  const order: string[] = snap.sheetOrder || Object.keys(snap.sheets || {});
  const activeId = order[0];
  const sheets: Record<string, any> = {};
  for (const sid of order) {
    const sh = snap.sheets[sid];
    sheets[sid] = sid === activeId ? sh : { ...sh, cellData: {} };
  }
  return { ...snap, sheets };
};

```ts
// 按需加载:把快照里某表的稀疏 cellData(带 style id)注入到已创建的工作表。
// 反解 style id -> 内联样式,确保 setValues 能正确套用样式(styles 为 bootstrap 全量返回的全局样式表)。
export function injectSheetCellData(sheet: any, cellData: any, styles: Record<string, any>) {
  const matrix: Record<string, Record<string, any>> = {};
  const rows = Object.keys(cellData || {});
  let maxR = 1, maxC = 1;
  for (const r of rows) {
    matrix[r] = {};
    const cols = Object.keys(cellData[r]);
    for (const c of cols) {
      const cell = cellData[r][c];
      const s = typeof cell.s === 'string' ? styles[cell.s] : cell.s;
      matrix[r][c] = s ? { ...cell, s } : { ...cell };
      maxR = Math.max(maxR, Number(r) + 1);
      maxC = Math.max(maxC, Number(c) + 1);
    }
  }
  sheet.getRange(0, 0, maxR, maxC).setValues(matrix);
}
```
