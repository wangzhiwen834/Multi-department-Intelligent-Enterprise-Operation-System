# 子项目 2:日期粒度聚合 + 数据大屏重构

## 背景与目标
- **现状**:`daily_metric.metrics` 已经由模板 `daily_ops` 存有一套与《经营报表》行对齐的丰富指标(营收/现金收入、客流拆分 会员/团购/散、钟数拆分 排/点/加、充值拆分 首充/续充/赠送、技师出勤/工资、新增会员、会员消费、足浴/SPA/小项业务收入、支付渠道 现金/抖音/美团/POS/支付宝/微信)。但 `Dashboard.vue` 只可视化其中一小撮,且只支持月粒度(`period=YYYY-MM`)。
- **目标**:
  1. 后端支持 日/周/月/年 四种粒度聚合 + 趋势下钻。
  2. 大屏按大河坎店《经营报表》真实维度重构可视化内容。
  3. 删除「任务仪表盘」(大圆盘 + 5 店任务表)。
  4. 为子项目 3 的 AI 抽取/可视化铺好数据契约:指标键与报表行对齐、聚合接口就绪(AI 只需把工作簿数据写进 `daily_metric`,大屏自动按粒度可视化)。

## 已确认设计决策
1. **粒度切换**:日/周/月/年,大屏自带切换器 + ‹ › 翻页 + 「回到今天」。Dashboard 自管日期状态,不再依赖顶栏全局月份选择器。
2. **趋势图语义(下钻到更细粒度)**:
   - 月 → 当月每日营收折线
   - 周 → 当周 7 天每日营收
   - 年 → 当年 12 个月营收柱
   - 日 → 往前 14 天每日营收(单天无内部细粒度,用 trailing 提供上下文)
3. **任务/时间进度**:大屏彻底移除。数据层 `shop.monthly_target` 与报表进度行保留(AI 仍可查),仅大屏不展示。免去周/日粒度下目标按比例拆分的复杂度。
4. **多店**:保留门店筛选 + 营收排名(按所选区间汇总)。指标维度取自大河坎报表,适用于所有足浴店。
5. **模板**:本子项目不扩模板(用现有指标键);模板/工作簿行扩展留给子项目 3。

## 技术要点
- **聚合源**:`daily_metric`(按日一行,JSONB `metrics`)。`reconciliation` 表的支付渠道指标(现金/抖音/美团/POS/支付宝/微信)已通过 sync 的 `metrics || $4` 合入同一日的 `metrics`,无需 join。
- **聚合规则**:加性指标 SUM;派生指标重算(`平均客单价 = Σ营收 / Σ客流`)。
- **区间计算**(pure JS date math,服务端 Node,Date 可用):
  - day → `[d, d]`
  - week → `[ISO 周一, 周日]`(PostgreSQL `date_trunc('week', d)` = ISO 周一)
  - month → `[月初, 月末]`
  - year → `[1/1, 12/31]`
- **趋势窗**:week/month/year = 区间内按更细一级分组(与 KPI 同一区间);day = `[d-13, d]` 逐日。
- **SQL**:`WHERE date BETWEEN $start AND $end`;趋势按 `date_trunc($unit, date)` 分组(unit 作 text 参数)。4 条查询:KPI+结构(一次 sum 多 key)、费用、排名、趋势。
- **向后兼容**:旧 `period=YYYY-MM` 仍可用 → 映射到 `granularity=month, date=period-01`。
- **粒度趋势 label**:day/week → `MM-DD`;month → `MM-DD`;year → `M月`。

## 新 payload 形状(替换现有 DashboardOverview)
```
{
  granularity: 'day'|'week'|'month'|'year',
  date: 'YYYY-MM-DD', rangeStart, rangeEnd, shopId,
  kpis: { revenue, customersTotal, avgCustomerPrice, rechargeTotal,
          totalClocks, newMembers, therapistAttendance, therapistWage, memberConsume },
  revenueTrend: [{ label, revenue }],
  customerStructure: { member, group, walkin },          // customers_member/group/walkin
  clockStructure: { arranged, requested, added },         // clocks_arranged/requested/added
  rechargeStructure: { first, renew, gift },              // recharge_first/renew/gift
  businessStructure: { footbath, spa, minor },
  paymentChannels: { cash, douyin, meituan, pos, alipay, wechat },
  expenseBySubject: [{ subject, amount }],
  shopRanking: [{ shopId, shopName, revenue }]            // 不再含 target/taskProgress
}
```

## 前端布局(Dashboard.vue 重写)
- **顶栏**:标题 + 区间标签 + 粒度分段控件(日/周/月/年)+ ‹ › + 「回到今天」+ 门店筛选 + 实时同步 pill。
  - 区间标签:day → `2026-07-15 周二`;week → `2026-07-14 ~ 07-20`;month → `2026 年 7 月`;year → `2026 年`。
  - ‹ › 步长:day ±1 天 / week ±7 天 / month ±1 月 / year ±1 年。
- **KPI 卡(2 行 × 4)**:营业收入、总客流、平均客单价、充值总额、总钟数、新增会员、技师出勤(人天)、技师工资。
- **图表行**:
  1. 营收趋势(line/bar,按子粒度)
  2. 客流结构(pie:会员/团购/散客)
  3. 钟数结构(stacked bar:排钟/点钟/加钟)
  4. 充值结构(pie:首充/续充/赠送)
  5. 业务结构(pie:足浴/SPA/小项)— 保留
  6. 支付渠道(pie)— 保留
  7. 费用 Top(bar)— 保留
  8. 门店营收排名(bar,可钻取选店)— 保留
- **删除**:任务仪表盘卡、5 店任务表、taskPct/timePct/behind/paceNote/gauge/taskRows、「任务完成」KPI 卡。
- 沿用 `--od-*` token + `THEMES['light']` + `Chart.vue` 组件。

## App.vue 改动
- `<Dashboard v-model:period="period" />` → `<Dashboard />`(自管日期,默认今天/当月)。
- 顶栏 `<input type="month">` 的 v-if 增加 `&& module !== 'dashboard'`(dashboard 模块隐藏全局月份选择器,避免与 Dashboard 内导航冲突)。Chat/ShopList 仍用顶栏月份,不受影响。
