# 工作表导出 / 导入(.xlsx)

## 目标
公司经营模块各店工作簿,支持:
- **导出**:把当前工作簿所有工作表导出成一个 `.xlsx` 文件,员工可在 Excel 打开查看 / 离线编辑。
- **导入**:把编辑好的 `.xlsx` 导回,**覆盖**当前工作表数据。

## 已确认决策
- 格式:.xlsx(SheetJS Community 版 `xlsx` 库,不依赖 Univer exchange)。
- 导入行为:覆盖现有。
- 范围:整个工作簿(日常录入 + 支出等多张表打包到同一个 xlsx 的多个 sheet)。
- 导出随时可用(查看态也行);导入需持锁(编辑态),因为要改数据。

## 技术方案

### 依赖
- `web/package.json` 加 `xlsx`(SheetJS Community,npm `xlsx@^0.18.5`)。

### 新文件 `web/src/sheet-io.ts`
封装导出 / 导入逻辑,保持 `univer.ts` 纯 Univer。

- `exportWorkbookToXlsx(fwb, template, filename)`:
  - 遍历 `template.sheets`,每张表用 `fwb.getSheetByName(label).getRange(0,0,400,cols).getValues()` 读二维数组(含表头)。
  - 裁掉尾部全空行。
  - `XLSX.utils.aoa_to_sheet` -> `book_append_sheet`(sheet 名 = label)。
  - `XLSX.writeFile(wb, filename)` 触发浏览器下载。
- `importXlsxToWorkbook(fwb, template, arrayBuffer)`:
  - `XLSX.read(data, { type:'array', cellDates:true })`。
  - 遍历 `template.sheets`,按 label 取 xlsx sheet;`sheet_to_json(header:1, raw:true)` 得 AOA。
  - **表头校验**:第 0 行应与 `template.columns[].label` 一致;不符则 `confirm` 警告"表头与模板不符,继续可能数据错位"。
  - **覆盖**:先 `getRange(1,0,400,cols).setValues(空矩阵)` 清空数据区,再 `getRange(0,0,n,cols).setValues(aoa)` 写入(含表头)。
  - 日期处理:AOA 中 `Date` -> `yyyy-mm-dd` 字符串(与现有 `extractForSync` 一致);其余原样。裁剪到 400 行上限。
  - 返回成功导入的表数。
- FRange 的 `getValues` / `setValues` 批量方法若不可用,降级为循环单格 `getValue/setValue`(实现时确认 API)。

### `web/src/views/Workbook.vue`
顶栏右侧加两个按钮:
- **导出**:随时可用。点击调 `exportWorkbookToXlsx(fwb, tpl, \`${shop.name}_${period}.xlsx\`)`。失败 `alert`。
- **导入**:仅 `editing` 时可用(需持锁),否则禁用并 title 提示"先点编辑"。点击触发隐藏 `<input type="file" accept=".xlsx">`。
  - `onchange` 读 `arrayBuffer` -> `importXlsxToWorkbook` -> 提示"已导入 N 张表,请检查后点「保存并同步」"。
  - **不自动保存**:让用户检查后再手动保存(避免错数据直接落库)。
- 复用现有 `.od-btn-ghost` 样式。

## 文件改动清单
1. `web/package.json` - 加 `xlsx` 依赖
2. `web/src/sheet-io.ts` - 新文件,导出 / 导入函数
3. `web/src/views/Workbook.vue` - 顶栏「导出」「导入」按钮 + 隐藏 file input + 调用逻辑

## 验证
1. `cd web && npm install`(装 xlsx),`npm run build` 通过。
2. 进某店工作簿 -> 点「导出」-> 得到 `店名_月份.xlsx`,Excel 打开能看到各表数据。
3. 在 Excel 改几个数 -> 「导入」选该 xlsx -> 画布数据被覆盖 -> 点「保存并同步」-> 大屏 / 台账数据更新。
4. 测试表头被改的 xlsx:导入时弹警告确认。
5. 非编辑态点「导入」:按钮禁用。

## 明确不做
- 不做 CSV(多表不适合)。
- 不做 Univer snapshot json 备份(用户选了 .xlsx)。
- 不做追加模式(用户选覆盖)。
- 不自动保存导入数据(用户手动保存,防错数据落库)。
- 不保留复杂样式 / 公式(SheetJS 社区版只保值;财务录入是值,够用)。
