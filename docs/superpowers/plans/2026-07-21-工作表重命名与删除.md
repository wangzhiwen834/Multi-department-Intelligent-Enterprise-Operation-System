# 工作表重命名 / 删除 / 新建 — 实现计划

日期:2026-07-21 · 分支:`feat/dashboard-granularity-redesign`

## 背景

用户要求:工作表要支持重命名(包括新建的和已存在的),还要支持删除。

经排查,**重命名 / 删除 / 新建工作表的 UI 能力其实已存在**——Univer 的原生底部工作表标签栏(SheetBar,随 `@univerjs/sheets-ui` 打包进 `UniverSheetsCorePreset`)已自带:
- 重命名:双击标签 / 右键「重命名」,内置校验(≤31 字符、首尾非 `'`、不含 `: [ ] : \ ? * /`、非空、不重名)。
- 删除:右键「删除工作表」,带确认弹窗;且原生禁止删除最后一张表。
- 新建:标签栏 `+` 按钮。
- 这些操作改的是内存态工作簿,持锁者点「保存并同步」后随快照持久化;查看态(`setEditable(false)`)被只读拦截——符合现有锁模型(只有持锁者能改结构)。

**真正的 bug**:同步逻辑 `extractForSync`(`web/src/univer.ts`)按**表名** `getSheetByName(s.label)` 匹配模板表。一旦用户重命名某张模板录入表(如「经营报表」),`getSheetByName('经营报表')` 返回 null,该表数据**被静默跳过**,大屏直接缺数且无任何报错。这会让用户误以为"重命名不被支持"。

删除模板表则是静默跳过(可接受:用户主动删除 = 该表不再录入);新建的自定义表不参与模板同步(正确:自由表不入大屏)。

## 决策(已与用户确认)

- UI 入口:**只用原生标签栏**,不加自定义顶栏菜单。
- 新建:**支持**(原生 `+` 按钮已有,无需额外代码)。
- 无论哪种,都先修同步 bug。

## 改动

### 1. 修复「重命名后大屏同步静默失效」(唯一代码改动)

文件:`web/src/univer.ts` — `extractForSync`

现状(约 50 行):
```ts
template.sheets.forEach(s => {
  const sheet = wb.getSheetByName(s.label);
  if (!sheet) return;
  ...
});
```

改为按 **sheetId** 优先匹配(模板表的 `id` 在建表时即设为 `s.key`——见 `buildFromTemplate` 第 26 行 `sheets[s.key] = { id: s.key, ... }` 与 `importXlsxToWorkbook` 第 275 行,Univer 重命名只改 `name` 不改 `id`,故 id 跨重命名稳定),表名回退(兼容旧快照 / 同名重建表):
```ts
template.sheets.forEach(s => {
  // 优先按 sheetId(=模板 key,重命名后不变)匹配,回退按表名(兼容旧快照/同名重建)
  const sheet = wb.getSheetBySheetId(s.key) ?? wb.getSheetByName(s.label);
  if (!sheet) return;
  ...
});
```

- `FWorkbook.getSheetBySheetId(sheetId): FWorksheet | null` 已确认存在(`node_modules/@univerjs/sheets/lib/types/facade/f-workbook.d.ts:182`)。
- `wb` 即 `fwb`(FWorkbook),原 `getSheetByName` 已在同对象上调用,签名兼容。
- 影响:重命名模板录入表后,同步仍按 `s.key` 提取数据 → `sheetKey` 正确 → 服务端 `syncWorkbook` 按 key 匹配 entry 列 → 大屏数据不丢。

### 2. 无需改动的部分(已验证)

- **后端**:`syncWorkbook` 收到的 `dailyMetrics` 已带 `sheetKey`(前端赋值),服务端按 key 查 `entryColsBySheet`,与表名无关 → 无需改。
- **快照持久化**:`performSave` → `fwb.save()`(整份快照,含所有表的 `sheets`/`sheetOrder`/`name`)→ `PUT /snapshot`。重命名/删除/新建均在快照内,重载 `createWorkbook(snapshot)` 即恢复。
- **懒加载**:`injectSheetCellData` 与 `GET .../sheets/:sheetKey/celldata` 在前端**无调用方**(Workbook.vue 直接用整份快照建表),不受影响。
- **导出**:`exportWorkbookToXlsx` 按 `sheetOrder` 遍历,自定义/重命名表均正常导出。
- **复制上月**:`headersOnlySnapshot` 保留每表第 0 行,自定义表表头一并复制(可接受)。

## 验证

1. `cd web && npx vue-tsc --noEmit && npm run build` 通过(类型 + 构建)。
2. 手动(用户):
   - 进入某店某月工作簿 → 点「编辑」取锁。
   - 重命名「经营报表」为「经营报表-测试」→ 录入一行数据 → 「保存并同步」→ 大屏该日数据**仍在**(修复前会消失)。
   - 右键删除一张非最后一张的表 → 保存 → 刷新后该表消失;大屏对应维度缺数(预期)。
   - 点 `+` 新建空白表 → 命名 → 保存 → 刷新后仍在;大屏不受影响。
   - 退出编辑后(查看态)右键标签:重命名/删除/新增均不可用(预期,只读拦截)。

## 不做

- 不加顶栏自定义「工作表」菜单(用户选原生标签栏)。
- 不限制可删除的表(用户要自由删除;原生已防删最后一张)。
- 不为重命名/删除做服务端审计日志(与现有「结构变更不入审计」一致;仅 `workbook.save` 入审计)。
